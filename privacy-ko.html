📚 ReadLens 개인정보 처리방침

최종 업데이트: 2025년 10월 11일

🔒 핵심 약속

ReadLens는 완전히 로컬 애플리케이션입니다. 모든 데이터는 사용자 기기에 저장됩니다. 저희는 사용자의 개인 데이터를 절대 수집, 저장 또는 전송하지 않습니다.

1. 어떤 데이터를 수집하나요?

간단한 답변: 없음

ReadLens는 어떠한 개인 데이터도 수집하지 않습니다. 모든 데이터는 사용자 기기에 로컬로 저장됩니다.

상세 설명

ReadLens는 다음 데이터를 사용자 기기에만 저장합니다:

  • 저장된 기사: 사용자가 저장하기로 선택한 웹 콘텐츠
  • AI 요약: API 키로 생성된 요약 (활성화된 경우)
  • 태그 및 메모: 기사에 추가한 태그와 개인 메모
  • 설정: 앱 환경설정 (테마, 언어 등)
  • API 키: AI 서비스 API 키 (로컬 암호화)

위의 모든 데이터는 사용자 기기에만 저장됩니다. ReadLens 팀은 이에 액세스할 수 없습니다.

2. 데이터를 어떻게 사용하나요?

ReadLens는 사용자 데이터를 사용하거나 액세스하지 않습니다. 이유는 다음과 같습니다:

  • ✅ 서버 없음: 데이터를 수집하는 백엔드 서버가 없습니다
  • ✅ 계정 시스템 없음: 등록이나 로그인이 필요하지 않습니다
  • ✅ 데이터 업로드 없음: 데이터가 기기를 벗어나지 않습니다
  • ✅ 제3자 추적 없음: Google Analytics나 추적 도구가 없습니다

3. AI 기능 및 API 키

🔑 BYOK (Bring Your Own Key) 모델

ReadLens는 "자신의 키를 가져오기" 접근 방식을 사용합니다. 이는 다음을 의미합니다:

사용자의 API 키

  • 암호화된 저장: API 키는 기기에 AES-256 암호화로 저장됩니다
  • 직접 연결: ReadLens는 콘텐츠를 AI 제공업체(OpenAI, Claude 등)에 직접 전송합니다
  • 중개자 없음: 중간 서버가 없으며, 요청을 보거나 기록할 수 없습니다
  • 사용자 제어: 언제든지 API 키를 삭제하거나 변경할 수 있습니다

AI 제공업체로의 데이터 전송

AI 요약 기능을 사용할 때:

  • 기사 콘텐츠가 브라우저에서 선택한 AI 제공업체로 직접 전송됩니다
  • 이 데이터는 ReadLens 서버를 거치지 않습니다 (서버가 없습니다)
  • 각 AI 제공업체의 개인정보 처리방침을 확인하세요:
    • OpenAI 개인정보 처리방침
    • Anthropic (Claude) 개인정보 처리방침
    • Google (Gemini) 개인정보 처리방침

4. Chrome 확장 프로그램 권한

ReadLens는 다음 Chrome 권한이 필요합니다:

activeTab (현재 탭)

목적: 기사를 저장하기 위해 보고 있는 웹페이지에서 콘텐츠를 읽습니다

데이터 대상: 로컬 기기에만 저장됩니다

storage 및 unlimitedStorage

목적: 기기에 기사, 설정 및 API 키를 저장합니다

데이터 대상: 로컬 Chrome 저장소에만 저장됩니다

contextMenus (컨텍스트 메뉴)

목적: 마우스 우클릭 빠른 저장 기능을 제공합니다

데이터 대상: 데이터 수집 없음

notifications

목적: 저장 성공/실패 알림을 표시합니다

데이터 대상: 데이터 수집 없음

https://*/*(네트워크 액세스)

목적: 웹페이지에서 콘텐츠를 추출하고 AI 제공업체 API에 연결합니다

데이터 대상: 선택한 AI 제공업체에만 전송됩니다 (AI 기능이 활성화된 경우)

5. 제3자 서비스

ReadLens 자체는 어떠한 제3자 서비스도 사용하지 않습니다. 그러나:

선택적 AI 서비스 (BYOK)

AI 요약 기능을 활성화하고 자신의 API 키를 제공하는 경우:

  • 기사 콘텐츠가 선택한 AI 제공업체로 전송됩니다
  • 이러한 제공업체는 자체 개인정보 처리방침이 있습니다
  • ReadLens는 이러한 전송을 보거나 기록하지 않습니다

사용하지 않는 서비스

  • ❌ Google Analytics
  • ❌ 광고 네트워크
  • ❌ 소셜 미디어 추적
  • ❌ 모든 형태의 사용 추적

6. 데이터 보안

로컬 암호화

  • API 키는 AES-256 암호화로 저장됩니다
  • 암호화 키는 브라우저에 저장됩니다
  • 누군가 기기 저장소에 액세스하더라도 복호화 키 없이는 API 키를 읽을 수 없습니다

입력 검증

  • 모든 사용자 입력은 검증 및 무해화됩니다
  • XSS(교차 사이트 스크립팅) 공격에 대한 보호
  • HTML 콘텐츠가 자동으로 무해화됩니다

7. 데이터 보존 및 삭제

데이터 보존

모든 데이터가 기기에 저장되므로:

  • 데이터는 사용자가 직접 삭제할 때까지 유지됩니다
  • 설정에서 언제든지 모든 데이터를 지울 수 있습니다
  • 확장 프로그램을 제거하면 모든 ReadLens 데이터가 자동으로 삭제됩니다

데이터 삭제 방법

  1. 개별 기사 삭제: 기사 목록에서 삭제 버튼을 클릭합니다
  2. 모든 데이터 지우기: 설정 → 데이터 관리 → 모든 데이터 삭제
  3. 완전 제거: Chrome에서 ReadLens 확장 프로그램을 제거합니다

8. 아동 개인정보 보호

ReadLens는 13세 미만의 아동을 대상으로 하지 않습니다. 저희는 아동으로부터 고의로 개인 데이터를 수집하지 않습니다 (사실, 누구의 데이터도 수집하지 않습니다).

9. 개인정보 처리방침 변경

이 개인정보 처리방침을 변경하는 경우:

  • 이 페이지 상단의 "최종 업데이트" 날짜를 업데이트합니다
  • 주요 변경 사항은 확장 프로그램 업데이트 노트에 공지됩니다
  • 최신 버전의 개인정보 처리방침을 언제든지 확인할 수 있습니다

10. 사용자 권리

ReadLens는 데이터를 수집하지 않으므로 기존의 정보주체 권리(액세스, 수정, 삭제 등)가 적용되지 않습니다.

그러나 사용자는 완전한 제어권을 가집니다:

  • ✅ 언제든지 로컬 데이터를 보고, 편집하거나 삭제할 수 있습니다
  • ✅ 언제든지 데이터를 내보낼 수 있습니다 (JSON, Markdown, CSV 형식)
  • ✅ 언제든지 AI 기능을 비활성화하거나 제거할 수 있습니다
  • ✅ API 키를 완전히 제어할 수 있습니다
